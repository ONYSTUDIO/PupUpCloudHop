# Game Rules — 안 떨어질개: 구름섬 점프

## 확정된 기본 룰

### 구름섬 이동
- 각 구름섬은 지정된 중심점(centerX, centerY)을 기준으로 원형 또는 타원형 궤도를 이동한다
- 궤도 반지름(orbitRadiusX, orbitRadiusY), 속도(orbitSpeed), 회전 방향(rotationDirection: 1 | -1)은 섬마다 다르게 설정된다
- 구름섬은 멈추지 않으며, 플레이어가 올라타 있는 동안에도 계속 이동한다
- 플레이어는 올라탄 구름섬과 함께 이동한다

### 점프
- 화면을 한 번 터치하면 플레이어가 점프한다
- 점프 방향은 가장 가까운 위쪽 구름섬을 자동으로 향한다
- 점프 중에는 중력이 적용된다
- 점프 중에는 출발 구름섬의 영향을 받지 않는다 (약 0.3초 이후 재착지 가능)

### 착지 판정
- 플레이어가 **아래 방향으로 떨어지는 중**에만 착지 판정이 발생한다
- 구름섬의 **윗면**에 닿았을 때만 착지로 인정한다
- 옆이나 아랫면에 부딪히면 착지가 되지 않는다
- 착지 시 플레이어는 구름섬 위에 붙어 함께 이동한다

### 점수
- 다른 구름섬에 새로 착지할 때마다 점수 +1
- 같은 구름섬에 다시 착지해도 직전 출발 구름이면 점수가 증가하지 않는다
- 최고 기록은 localStorage에 저장된다

### 게임 오버
- 플레이어가 화면 아래로 추락하면 게임이 종료된다
- 게임 오버 후 현재 점수와 최고 기록을 표시한다
- 다시 시작 또는 타이틀로 이동할 수 있다

## 검토 필요 항목

> 아래 항목은 아직 확정되지 않은 세부 룰입니다.

- [ ] **난이도 증가 방식**: 점수에 따라 구름섬 속도 증가? 간격 증가? 단계별 도입?
- [ ] **콤보 시스템**: 연속 착지 보너스 점수 도입 여부
- [ ] **장애물 등장 타이밍**: 몇 점 이후부터 장애물이 나타나는가?
- [ ] **장애물 종류**: 새, 번개, 강풍 등 — 충돌 시 즉사? 튕김?
- [ ] **아이템 시스템**: 점프 부스터, 보호막, 느린 구름 등
- [ ] **구름섬 생성 패턴**: 완전 랜덤? 손으로 설계한 패턴 세트?
- [ ] **구름섬 크기 변화**: 높이 올라갈수록 구름이 작아지는가?
- [ ] **카메라 공개 범위**: 다음 구름섬까지 미리 보이는 범위 조절
- [ ] **최고 점수 표시 방법**: 게임 중 HUD에 항상 표시 vs 게임 오버 시에만 표시
- [ ] **진동 피드백**: 착지 성공 / 게임 오버 시 진동 강도
